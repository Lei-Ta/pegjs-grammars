h1. reJSON

Grammar file of Enhansed JSON for PEG parser generator

The grammar adds QML-like features to JSON objects: 

* Keyless values
* Named objects
* Alternatives

reJSON can be used as replacement to XML and HTML, because it allows objects with same name and preserves order, and its syntax is simpler then JSON's one.

Below there is code in reJSON and parser's output in JSON: 

Input (reJSON):

bc. { 
  doctype { "html" }
  "html" {
    "head" { 
      "title" { "This is title"}
      "meta"  { "charset" : "utf-8"} 
    },
    "body" {
      "h1" { "id" : "pageheader"; "This is header" }
      "p"  { "This is paragraph" }
      "ul" { "li" [
         "One"
         "Two"
         "Three"
      ]} 
    } 
  }
}

Note: Names can be strings with quotes or symbols without quotes.

Ouput (JSON):

bc. { "$" : [ 
  { "name" : "doctype", "$" : [ "html" ] },
  { "name" : "html", "$" : [
    { "name" : "head", "$" : [
      { "name"    : "title", "$" : ["This is title"] },
      { "name"    : "meta", "charset" : "utf-8" } 
    ]},
    { "name" : "body", "$" : [
      { "name" : "h1", "id" : "title", "$" : [ "This is header" ] },
      { "name" : "p",  "$"  : [ "This is paragraph" ] },
      { "name" : "ul", "$"  : [
        { "name" : "li", "$" : [ "One"   ] },
        { "name" : "li", "$" : [ "Two"   ] },
        { "name" : "li", "$" : [ "Three" ] }
      ]} 
    ]} 
  ]}
]}
